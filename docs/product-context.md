# プロダクトコンテキスト

## ビジョン

**CreateLog = エンジニアだけの SNS / 交流アプリ**

エンジニアが投稿・フォロー・いいね・コメントで普通に交流する SNS。X はエンジニア以外も多くノイズだらけ、Zenn / Qiita は記事公開だけで交流できない。CreateLog はその空席を埋める「エンジニア専用の交流の場」。付随機能としてポートフォリオ管理 (マイプロダクト) と作業ログがある。

## ターゲット

- **メイン: 個人開発者** (本業/副業/フルタイム問わず、自分のプロダクトを作ってる人)
- **サブ: 全エンジニア** (フリーランス / 学生 / 起業初期 / vibe coder / 初心者 / 伸び悩んでる人)
- 最近 AI/vibe coding ブームで増えた「コード書き始めたばかりの人」「次のステップを探してる人」も歓迎

## 機能 (現在)

1. **SNS (core)**: 投稿・フォロー・いいね・コメント・返信。エンジニアだけが集まる
2. **ポートフォリオ管理**: 自分のアプリを登録・管理。リンク・スクショ・状態 (公開/開発中)・収益
3. **作業ログ (任意)**: 自分のアプリへの作業時間記録。VS Code / Cursor 拡張で自動収集
4. **属性別コミュニティ**: 学生 / フリーランス / vibe coder etc 同じ属性で交流

ユーザーはどれをメインに使ってもいい (SNS だけ、ポートフォリオだけ、作業ログだけ等)。

## 将来機能 (LP では現時点で言わない、約束しない)

- 仲間比較 (偏差値 / 上位 % / 同属性内ランキング) — 人は自分を他人と比べたい本能
- 企業向けマネタイズ: エンジニア人材データの販売、エンジニア採用プラットフォーム化

## 初動戦略

- 入口: X の個人開発者界隈。知名度を活かして 100〜1万 DL
- 勝ち筋: 機能は二番煎じでも UI/UX で凌駕して乗り換えさせる

## 収益モデル

1. クリエイター収益分配 (X 型): 全コンテンツ無料、広告収益から分配。Apple IAP 対象外
2. プラットフォーム広告: フィード・検索グリッドに広告カード
3. プレミアムサブスク: 広告削減、ログ見放題 (IAP 15% Small Business)
4. (将来) 企業向け: エンジニア活動データ販売、採用プラットフォーム

## App Store 規約の制約

- アプリカタログ表示はガイドライン 3.2.2(i) 違反。「ポートフォリオ」として扱う
- App Store 風 UI (アイコン一覧、レーティング、インストールボタン) は禁止
- 建て付けは「エンジニアの知見が集まる場所」
- UGC 機能には報告・ブロック・コンテンツフィルターが必須 (ガイドライン 1.2)
- 有料記事をやるなら Web のみ。アプリ内課金は不要
