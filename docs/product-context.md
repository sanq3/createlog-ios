# プロダクトコンテキスト

## ビジョン

**つくろぐ = エンジニアのLinkedIn + X + GitHub + ポートフォリオを一つに統合したアプリ**

- ターゲット: エンジニア全体（会社員、フリーランス、学生、個人開発者、バイブコーダー）
- 初動の入口: Xの個人開発者界隈。知名度を活かして100〜1万DL
- 勝ち筋: 機能は二番煎じでもUI/UXで凌駕して乗り換えさせる
- コアコンセプト: 「アカウント=ポートフォリオ」。作業記録するだけでポートフォリオが自動で育つ

## 収益モデル

1. クリエイター収益分配（X型）: 全コンテンツ無料。広告収益から分配。Apple IAP対象外
2. プラットフォーム広告: フィード・検索グリッドに広告カード
3. プレミアムサブスク: 広告削減、ログ見放題（IAP 15% Small Business）

## App Store規約の制約

- アプリカタログ表示はガイドライン3.2.2(i)違反。「ポートフォリオ」として扱う
- App Store風UI（アイコン一覧、レーティング、インストールボタン）は禁止
- 建て付けは「エンジニアの知見が集まる場所」
- UGC機能には報告・ブロック・コンテンツフィルターが必須（ガイドライン1.2）
- 有料記事をやるならWebのみ。アプリ内課金は不要
