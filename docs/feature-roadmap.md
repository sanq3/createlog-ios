## 設計決定ログ

決定事項をここに記録する。日付と理由を必ず書く。

### 2026-03-26: ホームフィードはカード型UI

- フラット（divider線のみ）とハイブリッド（背景色微差）を試した結果、カード型が一番良かった
- 将来の広告カード混在にもカード型が適している（LinkedIn式）
- 大手の多くはフラットだが、つくろぐは異種コンテンツ（ログ/投稿/記事/プロジェクト）が混在するためカード型が妥当

### 2026-03-26: 「◯人が作業中」はキャッシュ+集計方式

- Supabase Realtime Presenceはファンアウトコストが二乗で増加するため不採用
- Edge Function 5分cron → comparisons_cache → クライアント2-3分ポーリング
- コスト実質$0（Freeプランでも余裕）
- バックエンド接続フェーズで実装。UIモック段階ではハードコード

### 2026-03-26: 記事はMVPに含める

- 元々v2.1の予定だったが、検索タブのコンテンツとして必要
- MVP段階ではアプリ内簡易エディタ。Web版の本格エディタはv2.1以降
- 公開範囲: 全体公開 / フォロワー限定 / 下書き の3段階

### 2026-03-26: フィードのアルゴリズムはMVPでは時系列のみ

- Studyplusも時系列
- 「おすすめ」タブは将来追加（X式のFor You / Following切り替え）

### 2026-03-28: 投稿フロー設計

- FABはホーム画面のみに配置。全コンテンツ種類（投稿/記事/プロジェクト/コードスニペット/動画）をFABから作成可能
- テキスト投稿は280文字上限（X式）。画像は通常4枚、記事のみ10枚。動画は3分まで
- 投稿の編集は不可（削除のみ）。削除はいつでも可能
- 全コンテンツで下書き保存対応

### 2026-03-28: プロフィール編集・ハンドル設計

- ハンドル: 英数字+アンダースコア、3-15文字、30日に1回変更可能
- 内部リンクはユーザーIDベースで管理（ハンドル変更でリンクが壊れない）
- 自己紹介文: 150文字上限
- 外部リンク: 上限なし（WebサイトURL / GitHub / X / Zenn / Qiita等）
- アバター: カメラ撮影+ライブラリ選択

### 2026-03-28: レビューチケット制度設計（v2.1）

- **チケット制の相互レビュー**
  - レビューを1件書く → チケット1枚獲得
  - チケット1枚消費 → 依頼受付ONのユーザーにレビュー依頼可能
  - 新規ユーザーに初回1枚無料チケット付与（有効期限1ヶ月）。以降のチケットは期限なし
- **レビュー形式**: App Store式（5段階星評価 必須 + テキストコメント 任意）
- **フロー**: 依頼送信 → レビュアーが星評価を送信 → 自動完了 → チケット即付与
- **回答期限**: 3日。放置で信頼スコアが低下
- **拒否**: 依頼受付ONにしている以上、拒否不可。放置すると信頼スコア低下
- **依頼受付設定**: オンボーディングで設定。後から設定画面で変更可能
- **完了レビューはプロジェクトページに公開**。開発者からの返信可能
- **悪意あるレビュー**: 報告機能で対応
- **プロフィール表示**: レビュー実績数 + 信頼スコア
- **参考**: Critique Circle / Scribophile のクレジット型レビュー交換モデル

### 2026-03-28: コードスニペットのシンタックスハイライト

- 言語選択式（ユーザーが投稿時にドロップダウンで言語を選択）
- Highlightrライブラリ使用（highlight.jsのSwiftラッパー、WKWebView不要、190+言語対応）
- 業界標準: 開発者プラットフォームでハイライトなしは存在しない（GitHub/Discord/Slack/Qiita/Zenn全てハイライトあり）

### 2026-03-28: シェア・報告機能設計

- シェア: アプリ内リポスト（X式）+ 外部シェア（X/LINE等へのリンク共有）
- コンテンツ報告理由: スパム / 不適切 / ハラスメント / 著作権侵害 / なりすまし / その他（自由記述）の6種

### 2026-03-28: カテゴリ・カラーシステム設計

- **2層構造**: カテゴリ(親) → プロジェクト(子)。色はカテゴリが持ち、プロジェクトは親の色を継承
- **標準カテゴリ7個**(全ユーザー共通ブランドカラー): 開発, デザイン, 学習, ミーティング, ライティング, マーケティング, 事務
- **カスタムカテゴリ**: ユーザーが自由に追加。5色のカスタムパレットから選択
- **合計12色パレット**: 標準7色 + カスタム5色。Asset Catalogでライト/ダーク両対応
- **色だけで区別する**(アイコン不使用)。ソーシャルフィードで他人のレポートを見た時「この色=開発」と共通認識になる
- **プロジェクト単位で各色別表示**。同カテゴリ内の濃淡方式は不採用（区別しにくいため）
- **クロスプラットフォーム**: 色はHEX値でSupabaseに保存。iOS/Android/Web共通
- 参考: Toggl Track(14色パレット), Apple Screen Time(カテゴリ固定色), WakaTime(23活動タイプ)
